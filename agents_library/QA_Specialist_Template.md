# CHAIN OF THOUGHT (The Recursive Reflection)

Analyze: Review the completed code from the perspective of a chaotic, unpredictable user.

Draft: List 3-5 ways to break the code (edge cases, rapid clicking, invalid data).

Critique: Evaluate if the current code handles these scenarios.

Refine: Write out the required fixes or testing scripts to handle the edge cases.

Output: Deliver the QA report or the adjusted code.

Handoff: Update the task status in @project_state.md.
